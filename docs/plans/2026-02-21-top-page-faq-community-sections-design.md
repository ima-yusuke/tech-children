# トップページFAQ・コミュニティセクション追加 - デザインドキュメント

**作成日**: 2026-02-21
**対象**: resources/views/top.blade.php

## 概要

トップページ（top.blade.php）に、学習サイトとしての価値を高めるため、以下2つの新規セクションを追加する：

1. **FAQセクション**: よくある質問と回答を静的HTMLで提供
2. **コミュニティセクション**: サイト統計、ユーザーの声、SNSリンクを表示

## 要件

### 制約事項
- `<head>`タグ内は編集しない（SEO対策済み）
- 既存のデザインシステムと統一感を保つ
- レスポンシブデザイン対応

### 参考
- よくあるブログ記事・学習サイトのデザインパターン
- 既存のセクション（カテゴリ、最新記事、人気記事）のスタイルを踏襲

## 選択したアプローチ

**アプローチ1: シンプル縦積み配置**

既存のセクションと同じデザインパターンで、FAQ → コミュニティの順で縦に配置する。

**配置順序**:
1. ヒーローセクション
2. カテゴリセクション
3. 最新の記事セクション
4. 人気の記事セクション
5. **FAQセクション**（新規）
6. **コミュニティセクション**（新規）
7. フッター

**選定理由**:
- 既存セクションとの統一感
- 学習サイトとして自然な情報フロー
- レスポンシブ対応がシンプル

## デザイン仕様

### 1. FAQセクション

#### 構成
- セクションタイトル: 「よくある質問」
- アコーディオン形式のFAQアイテム（4-6個）
- クリックで開閉可能

#### デザイン詳細
- **背景色**: `#fff`（白背景）
- **パディング**: `80px 0`（既存セクションと同じ）
- **FAQアイテム**:
  - カード形式、ボーダー付き
  - 質問部分: クリック可能、太字
  - 回答部分: 初期状態は非表示、展開時に表示
  - 開閉アイコン: 「+」「-」で状態を表示
  - ホバーエフェクト: 既存のpost-cardと同様

#### FAQの内容例
1. Tech Childrenとは？
2. どのような記事がありますか？
3. 会員登録は必要ですか？
4. 記事はどのくらいの頻度で更新されますか？
5. おすすめの学習方法は？

#### 技術実装
- HTML5の`<details>`と`<summary>`要素を使用（CSS onlyアコーディオン）
- JavaScriptなしで動作
- スムーズな開閉アニメーションはCSSで実装

### 2. コミュニティセクション

#### 全体構成
- セクションタイトル: 「Tech Childrenコミュニティ」
- 3つのサブセクション:
  1. 統計情報エリア
  2. ユーザーの声エリア
  3. SNSリンクエリア

#### デザイン詳細
- **背景色**: `#fafafa`（既存のposts-section-altと同じ）
- **パディング**: `80px 0`

#### 2-A. 統計情報エリア

**構成**:
- サブタイトル: 「サイト統計」または統合タイトルのみ
- 3-4個の数値カード（横並び、グリッド）

**カード内容**:
1. 記事数（動的: Controllerから`$totalPosts`を渡す）
2. カテゴリ数（動的: `$categories->count()`）
3. 総閲覧数（動的: 全記事のview_countの合計）
4. 利用者数（静的または動的）

**スタイル**:
- カード形式、中央揃え
- 大きな数字（40px前後）+ 小さなラベル
- 背景白、ボーダー付き
- レスポンシブ: モバイルでは2カラムまたは1カラム

#### 2-B. ユーザーの声エリア

**構成**:
- サブタイトル: 「利用者の声」
- 2-3個のテスティモニアルカード（横並び、グリッド）

**カード内容**（例）:
1. 「プログラミング用語が難しかったのですが、このサイトのおかげで理解できるようになりました！」- 初学者Aさん
2. 「わかりやすい説明で、独学でも挫折せずに学べています。」- 学生Bさん
3. 「記事の質が高く、実務でも役立っています。」- エンジニアCさん

**スタイル**:
- 引用符アイコン（または装飾）
- 既存のpost-cardに似たカードスタイル
- イタリック体の引用文 + 通常フォントの名前
- 背景白、ボーダー付き

#### 2-C. SNSリンクエリア

**構成**:
- サブタイトル: 「SNSでフォロー」
- SNSボタン（横並び、中央揃え）

**ボタン内容**:
- Twitter（X）
- GitHub
- RSS
- その他（必要に応じて）

**スタイル**:
- 既存の`btn-outline`スタイルを活用
- アイコン + テキスト
- ホバー時に色変化

## データフロー

### Controllerの修正

`app/Http/Controllers/TopController.php`（または該当するController）に以下のデータを追加：

```php
$totalPosts = Post::published()->count();
$totalViews = Post::published()->sum('view_count');
```

### Viewへのデータ渡し

```php
return view('top', [
    'categories' => $categories,
    'recentPosts' => $recentPosts,
    'popularPosts' => $popularPosts,
    'totalPosts' => $totalPosts,      // 新規
    'totalViews' => $totalViews,      // 新規
]);
```

## スタイリング

### 新規CSSクラス

`resources/css/top.css`に以下を追加：

- `.faq-section`: FAQセクション全体
- `.faq-list`: FAQアイテムのリスト
- `.faq-item`: 個別のFAQアイテム
- `.faq-question`: 質問部分
- `.faq-answer`: 回答部分
- `.community-section`: コミュニティセクション全体
- `.stats-grid`: 統計情報のグリッド
- `.stat-card`: 統計カード
- `.testimonials-grid`: ユーザーの声のグリッド
- `.testimonial-card`: テスティモニアルカード
- `.social-links`: SNSリンクエリア
- `.social-btn`: SNSボタン

### デザイン原則

- 既存の配色（`#1a1a1a`, `#666`, `#999`, `#fafafa`, `#fff`）を踏襲
- フォント: 'Zen Kaku Gothic New'を使用
- トランジション効果: `0.3s ease`
- ボーダー: `1px solid #e5e5e5`
- ホバー時: `border-color: #999`, `transform: translateY(-2px)`

## レスポンシブ対応

### ブレークポイント

`@media (max-width: 768px)`:

- `.stats-grid`: 2カラムまたは1カラム
- `.testimonials-grid`: 1カラム
- `.social-links`: ボタンをスタック（縦並び）
- パディング調整: `56px 0`

## アクセシビリティ

- セマンティックHTML（`<section>`, `<article>`等）の使用
- `<details>`/`<summary>`でキーボード操作可能
- 適切な見出しレベル（`<h2>`, `<h3>`）
- リンクに適切なaria-labelを設定（SNSボタン）

## パフォーマンス考慮

- JavaScript不使用（可能な限り）
- CSSアニメーションは軽量
- 画像は使用しない（アイコンはCSSまたはSVG）
- 統計データはController側で効率的にクエリ

## 今後の拡張性

- FAQをデータベース管理に移行可能
- ユーザーの声を動的コンテンツ化可能
- SNSリンクを管理画面から編集可能
- 統計情報に新しい指標を追加可能

## 実装範囲外

- headタグの編集（SEO対策済みのため）
- 既存セクションの変更
- バックエンドロジックの大幅変更
